# README — Iteration 04 — All AI Types

## 1. Что было добавлено
Все 5 типов AI-пловцов, различимых формой/цветом/декором:
- **Linear** — красный круг, плывёт прямо, у препятствия разворачивается
- **Patrol** — оранжевый ромб, ходит по маршруту-петле из waypoint'ов (если клетка занята — ждёт ход)
- **Fast** — фиолетовый круг с двойной стрелкой, 2 клетки за ход (двигается по одной с промежуточной проверкой, «перепрыгнуть» игрока не может)
- **Delayed** — бирюзовый круг, двигается через ход; в ход отдыха притушен (alpha 0.45), перед ходом движения «просыпается»
- **Reactive** — жёлтый круг с глазом; если игрок на одной линии в пределах 3 клеток и без стен между — шагает К игроку, иначе ведёт себя как Linear

При поражении NPC-убийца делает punch-анимацию. Стрелки всех NPC показывают направление следующего шага.

## 3. Editor скрипты — какие меню запустить и в каком порядке
1. `SwimGame → Update Test Levels (Iteration 4)` — обновит TestLevel2 (Linear + Delayed + Patrol-прямоугольник) и TestLevel3 (Linear + Fast + Reactive). TestLevel1 не трогается. Внимание: затрёт твои правки этих двух раскладок

## 4. Как тестировать
1. Подставить TestLevel2 в Default Level у LevelLoader: оранжевый ромб ходит по прямоугольнику 6×3 в центре, бирюзовый Delayed в нижнем коридоре двигается через ход (видно по затуханию), красный Linear сверху
2. Подставить TestLevel3: фиолетовый Fast пролетает коридор по 2 клетки за ход (два прыжка подряд), жёлтый Reactive в нижнем коридоре — встань с ним на одну линию в пределах 3 клеток и он начнёт преследовать
3. Дать Fast догнать себя — убийца делает punch, проверка что Fast убивает и первым шагом, и вторым
4. Проверить Delayed: когда он притушен — следующий ход он стоит, можно безопасно пройти вплотную

## 5. Ожидаемый результат
Каждый тип читается мгновенно: форма+цвет+декор. Fast делает два последовательных прыжка (не телепорт). Delayed затухает после своего хода и светлеет перед следующим. Reactive поворачивает стрелку на игрока в момент обнаружения. Патруль никогда не сходит с маршрута, при блокировке другим NPC — ждёт.

## 6. Известные ограничения текущей итерации
- Точки патруля должны быть выровнены по осям (прямоугольные маршруты); по диагонали между точками пловец пойдёт сначала по X
- Reactive «теряет» игрока сразу при сходе с линии — без памяти/преследования за угол (намеренно, для просчитываемости)
- Walkthrough-решения уровней появятся в итерации 7 вместе с финальными 30 уровнями
