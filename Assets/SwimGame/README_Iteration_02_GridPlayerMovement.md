# README — Iteration 02 — Grid, Player, Movement

## 1. Что было добавлено
Клеточная система с загрузкой уровней из ScriptableObject (ASCII-раскладка), игрок с движением тапом по соседним клеткам, подсветка доступных ходов, камера с автоподгонкой под маленькие уровни и drag-скроллом для больших, HUD (название уровня, счётчик ходов, кнопки BACK и WAIT), всплеск-частицы и squash & stretch при движении.

## 2. Что изменилось с прошлой итерации
- **ЗАМЕНИТЬ файл** `Assets/SwimGame/Scripts/Managers/SoundManager.cs` — добавлен звук Deny (мягкий "отказ" при тапе по недоступной клетке)
- **УДАЛИТЬ файл** `Assets/SwimGame/Scripts/UI/GameSceneUI.cs` — заглушка из итерации 1 больше не нужна, её заменил GameHUD

- Сцена MainMenu не трогается

## 3. Editor скрипты — какие меню запустить и в каком порядке
1. `SwimGame → Create Test Levels (Iteration 2)` — создаст 3 тестовых уровня в `Assets/SwimGame/Levels/` (если уже существуют — не перезапишет)
2. `SwimGame → Build Game Scene (Iteration 2)` — пересоберёт сцену Game (поле, игрок, HUD, все ссылки). По умолчанию грузится TestLevel1_Small

Чтобы посмотреть другой тестовый уровень — перетащи его в поле `Default Level` компонента LevelLoader на объекте GameRoot (TestLevel3_Big проверяет скролл камеры).

## 4. Как тестировать
1. Открыть сцену MainMenu, Play, нажать PLAY — попадаешь на уровень
2. Тапать по подсвеченным (пульсирующим белым) соседним клеткам — игрок плывёт туда
3. Тапнуть по стене или дальней клетке — мягкий низкий звук отказа
4. Нажать WAIT — игрок "топчется" на месте, ход засчитывается
5. Подставить TestLevel3_Big в LevelLoader — поле не влезает в экран, драг пальцем/мышью скроллит камеру с плавным доездом, после хода камера сама доезжает к игроку у края экрана

## 5. Ожидаемый результат
Сине-голубое шахматное поле воды, тёмные стены, зелёная пульсирующая клетка выхода. Белый круглый игрок покачивается на воде (idle), при движении вытягивается по направлению и брызгает частицами, звук шага на каждый ход. Счётчик Moves в правом верхнем углу растёт с punch-анимацией. Подсветка ходов скрывается на время движения и обновляется после.

## 6. Известные ограничения текущей итерации
- NPC, победы и поражения нет — заплыть на выход пока ничего не даёт (итерация 3)
- WAIT просто тратит ход — World Phase появится в итерации 3
- Камера может частично перекрываться top bar'ом на впритык вписанных уровнях — отступы подберём при полировке
- Уровни тестовые, не из финальных 30

## 7. Что будет в следующей итерации
TurnManager (Player Phase → World Phase), первый NPC (Linear swimmer), победа на выходе, поражение при столкновении, попапы Win/Lose, Restart.
